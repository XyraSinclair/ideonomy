# A. P-1 + P-9 — oracle exists vs no oracle

**Composition value:** real, but only after adding a middle state.

The clean version is:

* **P-1:** target claim has an independent, executable truth-check.
* **P-9:** target claim has no legitimate scalar oracle; values/aesthetics/strategy remain plural.
* **Middle:** there are tests, proxies, samples, benchmarks, citations, stakeholder reads, or simulations that check *something adjacent* to the target.

The middle is not edge-case; it is the default for hard work.

Examples:

* A benchmark checks one operationalization of intelligence, not intelligence.
* Unit tests check implemented behavior, not total program correctness.
* A survey checks stated preferences, not actual welfare.
* A source citation checks textual support, not global truth.
* A backtest checks historical performance, not future robustness.
* A judge rubric checks visible quality, not latent downstream value.

So the dichotomy is not exhaustive unless P-1 is defined very strictly as **oracle for the actual target proposition**, not “some executable check.”

**Default-LLM differentiation:** partially real.

A strong LLM will often distinguish objective and subjective questions when prompted. What it usually does **not** do by default is precommit to an expected oracle result before answering, then run the oracle, then refuse to collapse no-oracle questions into fake numbers. So P-1 and P-9 are differentiated as procedures, not as concepts.

**Failure/mislead points:**

1. **Proxy-oracle laundering.** The model treats a benchmark, citation, poll, or scoring rubric as if it decides the real target.
2. **False P-1 routing.** Once routed to P-1, the model may skip P-9 entirely, even though the real issue is normative.
3. **Oracle endogeneity.** The “oracle” is produced by the same reasoning path it is supposed to check.
4. **Metric capture.** The test becomes the target.
5. **Underspecified target.** The oracle is executable but the proposition being checked is not the actual claim users care about.

**Fix:** add an explicit third category:

> **Exact oracle / proxy oracle / no oracle.**

The proxy state should trigger something like: P-11 to decompose the target, P-1 only on exact subclaims, P-9 on irreducible axes, and P-7 when the proxy is gameable.

# B. P-9 + P-10 — triangulation harness + residue ledger

**Composition value:** real.

P-9 produces unusually structured residue: contested axes, spreads, independent judgments, and owner-calls. P-10 makes that residue cumulative across sessions. Neither does the whole job alone.

* P-9 alone gives honest no-oracle handling for one session.
* P-10 alone can persist arbitrary notes, including junk.
* Together, they create **cumulative non-resolution** instead of repetitive churn.

That is genuine compounding.

**Default-LLM differentiation:** real, especially because default chat systems are amnesic/episodic. Even when they summarize unresolved issues, they usually do not require session N+1 to cite session N’s unresolved residue before expanding.

**Failure/mislead points:**

1. **Ledger ossification.** Old contested axes become sacred even after the frame changes.
2. **Ritual citation.** Session N+1 cites the residue but does not actually use it.
3. **Residue hoarding.** The ledger accumulates open questions faster than it resolves or retires them.
4. **Stale judgment spread.** Independent reads from one context get reused after stakeholders, goals, or facts change.
5. **Ambiguity debt.** “Contested axis” becomes a polite way to avoid decision.

**Keep condition:** P-10 must store typed residue, not prose summaries. For P-9, that means at minimum: axis, judgment spread, source of each read, owner of irreducible call, decision status, and retirement condition.

# C. P-3 + P-6 — reframe vs coin missing concept

**Composition value:** conditional.

There is a real distinction, but it is fragile.

* **P-3** changes representation: altitude, basis, domain, coordinate system, analogy class.
* **P-6** changes ontology: it introduces a new unit of thought with a membership test.

P-6 is a special case of P-3 if P-3 is allowed to mean “any compression-producing transformation.” Under that broad reading, P-6 is slop.

P-6 deserves independence only if it has a narrower trigger:

> repeated awkward circumlocution across a corpus indicates a stable missing category; the coined concept must reduce description length and predict a previously unlisted case.

That makes it closer to concept discovery than reframing.

**Default-LLM differentiation:** weak-to-medium.

Strong LLMs already reframe and coin labels constantly. The differentiation is not “invent a name”; it is the executable gate: membership test, compression delta, and predictive reach.

**Failure/mislead points:**

1. **Jargon inflation.** A new noun creates the feeling of explanation.
2. **MDL spoofing.** A vague term shortens text while destroying precision.
3. **Reification.** A linguistic convenience gets mistaken for a real structure.
4. **Premature abstraction.** The model coins the concept before exhausting simpler decompositions.
5. **Semantic drift.** The concept compresses the original corpus but fails on new cases.
6. **Aesthetic compression.** The description becomes elegant but less predictive.

**Verdict:** P-6 is keepable, but only with a hard “no new noun without membership test + held-out prediction” rule. Otherwise merge it into P-3.

# D. P-5 + P-11 — alternatives + coverage denominator

**Composition value:** strong.

They solve different halves of the same failure.

* **P-11** asks: what is the full denominator of cases/sources/branches?
* **P-5** asks: among live alternatives, what observation would separate them?

P-11 prevents the classic LLM move of listing four convenient possibilities while ignoring the actual space. P-5 prevents the opposite failure: exhaustive taxonomy with no discriminating tests.

Together they create a real diagnostic engine:

1. State the denominator.
2. Label every branch.
3. Generate hypotheses.
4. Attach discriminating predictions.
5. Refuse convergence while multiple hypotheses explain all observed facts.

**Default-LLM differentiation:** real.

A strong LLM may list alternatives. It usually will not prove the denominator, mark every branch covered/replayable/gap/ruled-out, and forbid convergence while confirmed predictions are shared.

**Failure/mislead points:**

1. **Open-world denominator.** In many domains, the full set of possibilities is unknowable.
2. **False exhaustiveness.** The denominator looks complete because it is neatly formatted.
3. **Non-independent hypotheses.** The alternatives overlap or differ only verbally.
4. **Mixed causality.** “Exactly one survives” fails when multiple hypotheses are jointly true.
5. **Expensive discrimination.** The decisive test may be impossible, unethical, or too slow.
6. **Denominator mismatch.** The listed denominator covers documents, but the real uncertainty is causal; or covers causal branches, but the real issue is incentives.

**Fix:** allow outcomes like “multiple jointly survive,” “underdetermined,” and “denominator open-world.” Do not force uniqueness when the world is compositional.

# E. P-7 + P-9 — incentives + values

**Composition value:** real and important.

P-9 should come before P-7 when the objective is not settled.

* P-9 says: do not fabricate a scalar value function.
* P-7 says: once the objective/constraint is chosen, design the mechanism so the desired behavior is an equilibrium.

The real compounding is this:

> P-9 prevents P-7 from optimizing the wrong thing; P-7 prevents P-9 from remaining inert pluralism.

That is valuable.

**Default-LLM differentiation:** real if P-7 is actually formal.

Strong LLMs often discuss stakeholders, tradeoffs, and incentives. They rarely model the metric-gamer as an adversary and verify “no profitable unilateral deviation.” That is the differentiator.

**Failure/mislead points:**

1. **Utility laundering.** P-9 finds plural values; P-7 collapses them into one fake objective anyway.
2. **Power blindness.** “Stakeholder values” may hide coercion, bargaining power, or institutional constraints.
3. **Bad rationality model.** The best-response proof assumes agents optimize the modeled payoff.
4. **Proxy capture.** The mechanism rewards a measurable stand-in for the value.
5. **Multiple equilibria.** Desired behavior may be an equilibrium but not the selected one.
6. **Normative evasion.** The mechanism designer claims neutrality while encoding value choices in constraints.

**Keep condition:** P-7 must be explicitly conditional on a P-9-owned value choice when values are irreducible. No “the mechanism says this is optimal” unless the value owner has chosen the axis/tradeoff.

# F. P-1 + P-2 + P-4 — verification stack

**Verdict:** keep the stack; do not pretend the three are equally independent.

They are all verification moves. The difference is artifact-type and timing:

| Skill   | Distinct artifact                   | Failure it catches                                                       |
| ------- | ----------------------------------- | ------------------------------------------------------------------------ |
| **P-2** | Smallest nontrivial worked instance | Mechanism misunderstanding; overgeneralization; hidden base-case failure |
| **P-1** | Precommitted independent oracle     | Backward rationalization; post-hoc fitting; unverifiable confidence      |
| **P-4** | Blind adversarial attack suite      | Fragility, counterexamples, hidden assumptions, motivated defense        |

So the distinction is real, but they are not three separate “premier” ideas at the same abstraction level. P-1 is the strongest and most non-default. P-2 and P-4 are valuable because they force execution, but they are closer to what good reasoners already attempt.

**Default-LLM differentiation:** mixed.

* P-2: strong LLMs often try examples, but not reliably the **smallest nontrivial** one that exercises the mechanism.
* P-4: strong LLMs often self-critique, but usually not blind, not adversarial enough, and not execution-gated.
* P-1: strong LLMs almost never precommit expected oracle outputs before answering.

**Failure/mislead points:**

1. **Toy-case deception.** The smallest case works but the mechanism fails at scale.
2. **Phase transition miss.** The nontrivial behavior begins later than the chosen small case.
3. **Oracle dependence.** The oracle is not independent of the answer.
4. **Adversary collusion.** Same-model adversary shares the same blind spots.
5. **Attack underspecification.** “Serious attack” becomes subjective.
6. **Verification theater.** All three gates fire on a narrowed or wrong target.
7. **Proof-by-example.** P-2 is mistaken for general proof.

**Kill or keep?**

Keep **as a named protocol**, but I would not preserve all three as equally independent premier skills if you are aggressively de-slopping the library.

Best packaging:

> **Verification stack:** instantiate the smallest mechanism case; precommit oracle/expected result; then blind-red-team the final answer with executable attacks.

Inside that stack, P-1 remains the core skill. P-2 and P-4 are mandatory phases/subskills.

# G. P-0 + P-9 + P-10 — route → no-oracle judgment → persist residue

**Composition value:** real, but narrower than “operational core.”

This loop works for no-oracle work:

1. P-0 detects that a hard oracle is unavailable.
2. P-9 dimensionalizes the value space and refuses fake scalarization.
3. P-10 persists the unresolved axes, spreads, and owner-calls.

That is a coherent operating loop.

But it is not enough as the central router for the whole library, because it is biased toward soft/no-oracle handling. The dangerous misroute is: “this seems subjective,” when actually it contains hard factual, structural, or incentive subclaims that should be split out.

**Default-LLM differentiation:** real if implemented.

Default LLM behavior does not reliably route, log misroutes, or persist unresolved axes across sessions.

**Failure/mislead points:**

1. **P-0 becomes unfalsifiable.** A meta-router can always rationalize its dispatch after the fact.
2. **P-9 overuse.** Hard subclaims get softened into “contested axes.”
3. **P-10 churn.** The ledger remembers unresolved material without creating progress.
4. **No closure pressure.** There is persistence but not necessarily resolution, retirement, or escalation.
5. **Misroute laundering.** “Routed to P-9” becomes a way to avoid harder P-1/P-5/P-11 checks.

**Fix:** the stronger core is not P-0 + P-9 + P-10. It is closer to:

> P-0 + P-11, then route each labeled branch to P-1, P-5, P-7, P-8, or P-9, with P-10 storing only unresolved residue.

P-11 is the missing stabilizer because it forces decomposition before the router chooses “no oracle.”

# H. P-3 + P-8 + P-6 — reframe / localize-glue / coin concept

**Composition value:** real, but high-risk.

These are distinct structural operations:

* **P-3:** choose a representation where the problem becomes easier.
* **P-8:** solve locally and test whether local solutions glue.
* **P-6:** when overlap residue recurs, coin the missing invariant/concept.

The strong version is:

1. Reframe the problem into a domain with tractable patches.
2. Solve each patch.
3. Compute overlap discrepancies.
4. If discrepancies persist, name the obstruction or coin the missing concept.
5. Use that concept to either glue or prove non-gluability.

That is not something default LLMs do reliably. It is especially strong for architecture, theory-building, taxonomy design, mathematical analogy, and multi-domain synthesis.

**Default-LLM differentiation:** conditional.

Strong LLMs love reframing and naming. They do not reliably compute overlaps, identify obstructions, or enforce membership tests. So the differentiation lives almost entirely in the gates.

**Failure/mislead points:**

1. **Abstraction cosplay.** The model names invariants without calculating discrepancies.
2. **Wrong cover.** Local patches do not cover the actual domain.
3. **Decorative obstruction.** The “named invariant” is merely a label for failure.
4. **Invalid transport.** A solved-domain analogy imports structure that the target lacks.
5. **Compression illusion.** The reframe is shorter because it hides terms.
6. **Concept overfitting.** P-6 coins a term that compresses the current examples but fails out-of-sample.
7. **Gluing on wrong overlaps.** The local solutions agree where checked but diverge elsewhere.

**Keep condition:** pair H with P-11 when stakes are high. P-8 without coverage proof can produce beautiful local correctness over a non-covering atlas.

# Across the whole set

## 1. Weakest / most likely slop

### 1. **P-0 — route-to-the-right-move**

Most dangerous as slop because meta-routing is easy to narrate and hard to verify. “Chosen skill’s gate fires” is not enough, because the wrong skill can still produce a satisfying local artifact.

Failure signature: the router sends a partially checkable problem to P-9, or sends a proxy-oracle problem to P-1, then congratulates itself because some gate fired.

P-0 needs a routing table, misroute taxonomy, and confusion log. Otherwise it is executive-function prose.

### 2. **P-3 — reframe-until-it-dissolves**

This is powerful, but it is also something strong LLMs already attempt by default: analogy, abstraction, basis shift, simplification. The MDL gate helps, but it is gameable because shortness can hide lost constraints.

Failure signature: the reframed statement is shorter because it is vaguer.

P-3 needs a semantic-preservation check, not just a compression check.

### 3. **P-6 — coin-the-missing-concept**

This is the most LLM-native failure mode in the set. Models love coining tidy labels. A new term can make a mess feel solved.

P-6 survives only if the membership test is crisp and the concept predicts held-out cases. Otherwise it is a special case of P-3 with extra jargon.

Near-miss: **P-4** is also default-ish if it means “critique my answer.” It becomes differentiated only when blind, adversarial, and execution-gated.

---

## 2. Genuinely differentiated from strong default LLM behavior

### 1. **P-1 — build-the-oracle-before-the-answer**

The precommitment is the key. Strong LLMs verify after answering; they rarely commit an expected oracle result before constructing the answer. This directly attacks rationalization.

### 2. **P-7 — design-the-mechanism-from-the-desired-equilibrium**

Most LLMs discuss incentives verbally. They do not normally formalize the metric-gamer as an adversary and verify no profitable unilateral deviation. This is genuinely non-default if machine-checked.

### 3. **P-10 — carry-the-residue-forward-across-sessions**

This is differentiated infrastructurally. Default LLM workflows lose unresolved residue or summarize it weakly. A typed ledger with a next-session citation gate creates actual cumulative cognition.

Honorable mention: **P-11**. Strong LLMs may list coverage when asked, but they do not usually prove the denominator and mark every element covered/replayable/gap/ruled-out.

## 3. Missing high-value pair/triplet

### Missing triplet 1: **P-11 + P-1 + P-9**

This is the fix for the dangerous middle in A.

Use P-11 to decompose the target into subclaims. Then route each subclaim:

* exact oracle → P-1
* proxy-only check → label as proxy, do not overclaim
* irreducible value/aesthetic/strategic axis → P-9
* unlabeled branch → not done

This would prevent oracle/no-oracle false dichotomy.

### Missing triplet 2: **P-5 + P-1 + P-4**

This is the strongest diagnostic tournament.

* P-5 enumerates live hypotheses and discriminating predictions.
* P-1 precommits expected test results before answer formation.
* P-4 attacks the surviving hypothesis blind.

This is much stronger than generic “consider alternatives.”

### Missing triplet 3: **P-11 + P-8 + P-3**

This is the structural safety version of H.

* P-11 proves the cover/denominator.
* P-8 solves patches and checks overlaps.
* P-3 supplies the representation in which the cover is natural.

This prevents beautiful local-gluing work from silently missing regions of the problem.

### Missing pair: **P-7 + P-4**

Mechanism design needs adversarial red-team by default.

P-7 proves desired behavior is a best response under the modeled game. P-4 should then search for unmodeled deviations, side channels, collusion, sybil behavior, metric gaming, and equilibrium selection failures.

## 4. Single sharpest critique of the whole framing

The framework risks **gate laundering**.

An executable gate prevents vacuity only if the gate is attached to the real target. Many of your gates can fire on a surrogate artifact:

* an oracle for the wrong claim,
* a smaller case that misses the phase transition,
* an MDL-shorter reframe that drops constraints,
* a coined concept that compresses language but not reality,
* a mechanism equilibrium under the wrong utility model,
* a coverage denominator that omits the true branch,
* a residue ledger that preserves yesterday’s wrong frame.

So the central missing invariant is:

> **target preservation across moves.**

Every skill changes the problem somehow: tests it, reframes it, localizes it, names it, mechanizes it, dimensionalizes it, or persists it. The sharpest failure is not that the move is non-executable. The sharpest failure is that the move executes cleanly on a transformed target that is no longer the one that mattered.

Your strongest next refinement would be a universal pre-gate:

> Before any skill fires, state what target property must be preserved; after the skill fires, verify that the artifact still binds to that property.

Without that, the library can produce high-assurance wrongness: beautifully verified cognition aimed one degree away from the real problem.
